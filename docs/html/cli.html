Headless workflow

Render and automate `.opencomp` scripts without the browser

The CLI loads OpenComp script files, runs the same Python scripting layer as the in-app Script Editor, and renders Write nodes through the backend evaluator. 

python -m opencomp.cli shot.opencomp \
  --render Write1 \
  --range 1001-1005 \
  --json

Core Commands

Save

--save shot_v002.opencomp

Inspect

--list-nodes --list-scripts

Render

--render Write1 --range 1001-1010

Script

--run-script setup.py --save

Metadata

--metadata Read1 --range 1001

Nuke

--export-nuke shot.nk

Range Syntax

SyntaxMeaning
1001One frame.
1001-1005Inclusive frame range.
1001,1003,1005Explicit frame list.
1001-1010x2Stepped inclusive range.

Script Editor Compatibility

read = opencomp.node("Read1")
read.value("path").setValue(r"E:\plates\shot_####.exr")

grade = opencomp.create_node("Grade", name="Grade2")
grade.value("gain").setValue(1.15)
grade.setInput("in", read)

root.value("name").setValue("shot_comp")

Current Limits

Write nodes only

Headless render intentionally renders Write nodes, matching compositor render semantics.

Process-local cache

The CLI uses evaluator cache only for the current command process.

V1 `.nk`

Common OpenComp nodes export as native Nuke blocks; unsupported nodes become labeled NoOp fallbacks.
