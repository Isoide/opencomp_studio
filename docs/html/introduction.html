Introduction

A Nuke-like compositor foundation in the browser

OpenComp Studio currently combines a React canvas UI with a FastAPI image processing backend. The goal is to make node-based compositing, OCIO viewing, EXR metadata, scripting, and caching available in a local web app.

Main Panels

Node Menu

Categorized node list for I/O, color, channel, merge, transform, metadata, and output nodes.

Viewer

Canvas viewer with display/view/channel controls, proxy/full-res, pixel readout, compare tools, and playback.

Graph

Canvas node graph with top-to-bottom flow and detachable connections.

Inspector

Node properties, root settings, metrics, and log views.

Script Editor

Runs Python-like scripts against the backend session to create and modify graphs.

Preferences

Project-level cache, hotkey, proxy, tile, and startup customization settings.

Typical Flow

1. Read
Load plate or render sequences.
2. Process
Grade, reformat, shuffle, merge.
3. View
Connect to Viewer numbered inputs.
4. Cache
Scrub or play while caches warm.
5. Write
Render through a Write node.

Current State

Ready for prototype slapcomps

Read multiple EXR sequences, merge them, apply basic color transforms, preview through OCIO, and write EXR output.

Still a v1 engine

Graph processing is CPU-based, caches are in-memory, and true visible-region tile rendering is a future step.
