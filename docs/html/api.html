API

Control the app through HTTP, WebSockets, and the Python script editor

The backend exposes project, graph, color, viewer, cryptomatte, cache, render, and scripting endpoints. The Script Editor uses the same backend session API to build nodes and change root settings. 

POST /api/python/run
GET  /api/graph
POST /api/viewer/frame
WS   /ws/viewer/float
GET  /api/cache/status

Project Files

OpenComp project scripts use the .opencomp extension. The API can save/load those files and can write a v1 .nk export with native node blocks and stack connections. The browser UI can also import selected local .opencomp files through /api/projects/import. 

POST /api/projects/save
POST /api/projects/load
POST /api/projects/import
POST /api/projects/export-nuke
POST /api/projects/export-nuke/content

Script Editor Example

read = opencomp.create_node("Read", name="Read1", path=r"E:\plates\shot_####.exr")
grade = opencomp.create_node("Grade", name="Grade1", gain=1.2)
viewer = opencomp.create_node("Viewer", name="Viewer1", active_input="0")

grade.setInput("in", read)
viewer.setInput("0", grade)

root.value("frame_start").setValue(1001)
root.value("frame_end").setValue(1010)

Float Viewer Stream

The preferred viewer path is a WebSocket frame stream that sends a JSON header followed by binary float tiles. The frontend uploads the frame to WebGL2 and applies display processing there. 

{
  "node_id": "Viewer1",
  "frame": 1001,
  "channel": "rgba",
  "precision": "float16",
  "stream_tiles": true,
  "tile_height": 128
}
